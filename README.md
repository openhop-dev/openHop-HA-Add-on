## About

This repository publishes two Home Assistant add-ons around the official openHop Repeater container images. Both channels keep Repeater configuration and runtime state in Home Assistant-managed persistent storage while exposing the upstream web interface on port `8000`.

The add-ons use the image selected by the installed channel. They do not download or switch application branches at startup.

## Choose a channel

| Add-on | Upstream image | Best for |
|---|---|---|
| **[openHop Repeater Main](openhop_repeater_main/)** | `openhop/openhop-repeater:main` | Mainline updates with a lower rate of change |
| **[openHop Repeater Dev](openhop_repeater_dev/)** | `openhop/openhop-repeater:dev` | Testing the newest upstream development changes |

The channels are separate Home Assistant add-ons with different slugs and separate configuration/data directories. Installing one does not replace or migrate the other.

## Highlights

- Continuous MeshCore-compatible LoRa packet forwarding and routing on a Home Assistant host
- Browser-based setup, configuration, logs, monitoring, and API access on port `8000`
- Real-time packet statistics, neighbor discovery, signal data, and historical metrics
- Native SX1262 SPI, CH341 USB-to-SPI, pyMC TCP/USB modem, and KISS serial radio backends
- MQTT, GPS, external sensors, companion identities, and optional [openHop Glass](https://github.com/openhop-dev/openHop-Glass) integration
- Companion Home Assistant integration for Repeater telemetry and controls
- Separate, predictable `main` and `dev` image channels
- Native `aarch64` and `amd64` container images for low-power and x86 Home Assistant hosts
- Persistent configuration, node identity, API-token database, and runtime state
- Unique first-start credentials plus conservative configuration upgrades
- Cold Home Assistant backups and clean process lifecycle handling
- Direct SPI, GPIO, USB, serial, host-network, and dynamic companion-service access

## Architecture support

The published `openhop/openhop-repeater:main` and `:dev` image manifests currently include:

| Home Assistant architecture | Container platform | Supported |
|---|---|---|
| `aarch64` | `linux/arm64` | Yes |
| `amd64` | `linux/amd64` | Yes |

No other Home Assistant architectures are declared by these add-ons.

## Installation

[![Open your Home Assistant instance and add this repository.](https://my.home-assistant.io/badges/supervisor_add_addon_repository.svg)](https://my.home-assistant.io/redirect/supervisor_add_addon_repository/?repository_url=https%3A%2F%2Fgithub.com%2Fopenhop-dev%2FopenHop-HA-Add-on)

Or add the repository manually:

1. Open **Settings → Add-ons → Add-on Store**.
2. Open the top-right menu and select **Repositories**.
3. Add:

   ```text
   https://github.com/openhop-dev/openHop-HA-Add-on
   ```

4. Choose **openHop Repeater Main** or **openHop Repeater Dev**.
5. Review the hardware-access warning, install the add-on, and follow its channel-specific README.

## First start

On first start, the add-on creates a complete `/config/config.yaml` from the template bundled in the selected upstream image. Shared example credentials are replaced with unique values before the file is published.

The add-on also retains:

| Container path | Purpose |
|---|---|
| `/config/config.yaml` | Repeater configuration and generated credentials |
| `/config/identity.key` | Node identity when generated by openHop Repeater |
| `/var/lib/openhop_repeater` | Database, authentication state, and runtime data |

Start the add-on once, stop it, then edit the generated configuration in the add-on's private configuration directory before attaching radio hardware.

## Hardware access and security

> [!WARNING]
> These add-ons request `full_access`, disable AppArmor, and use host networking so openHop Repeater can access GPIO, SPI, USB, serial hardware, and companion services. Install only trusted images and configuration, and expose the web/API ports only on a trusted network or VPN.

Home Assistant's **Protection mode** must also be disabled when local GPIO or SPI access is required.

## Home Assistant integration

To monitor and control the running Repeater from Home Assistant, install the companion [openHop Repeater Home Assistant integration](https://github.com/openhop-dev/openHop-HA-Integration).
